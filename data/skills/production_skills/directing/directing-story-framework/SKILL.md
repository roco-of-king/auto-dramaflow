# "很久很久以前"故事构建框架

## R — 原文 (Reading)
> 很久很久以前……有一位……他/她……但是……有一天，事情发生了……因此，接下来……但与此同时……为……所不知的是……直到……的时候……突然，出乎意料的是……结果是……从这以后……
> — Mackendrick, 第9章, ~p.64

## I — 方法论骨架 (Interpretation)
这是一个用日常口语连词引导的逐步叙事构建工具。它的核心思想是：故事不是靠抽象的"三幕结构"或"节拍表"来搭建的，而是靠一系列因果推进的连词来驱动的。

框架的每个步骤对应戏剧结构的一个关键要素：设定时间地点和主角（很久很久以前/有一位）、引入障碍（但是）、发动攻击点（有一天/事情发生了）、因果推进（因此/接下来）、并行叙事线（但与此同时）、信息差操控即戏剧反讽（为……所不知的是）、高潮前的逼近（直到……的时候）、突转（突然/出乎意料的是）、冲突解决（结果是）、最终结局（从这以后）。

这个工具的独特价值在于：它用人类讲故事的自然语言节奏来组织结构，比抽象标签更容易操作。任何步骤的缺失都意味着结构缺陷。比如没有"但是"，说明主角没有障碍；没有"为……所不知的是"，说明缺少反讽张力。

## A1 — 书中的应用 (Past Application)
### 案例 1: 所罗门王故事的剧本改编（c21）
- **问题**: 《圣经》中的所罗门王故事是一个简短的审判轶事，如何将其扩展为有完整戏剧弧线的三场戏剧本？
- **方法论的使用**: 麦肯德里克用故事构建框架逐步扫描：确定主角（年轻的所罗门而非老国王）、设定障碍（假冒母亲的欺骗）、设计攻击点（第一个女人的控诉）、安排并行发展（三个法官代表不同怀疑态度）、埋入反讽（观众知道所罗门的计谋而假冒母亲不知道）、导向突转（用剑劈婴儿的威胁）和结局（老臣们态度的转变）。
- **结论**: 框架的每个步骤都对应可操作的剧本设计决策，即使是高度简略的原始素材也能通过此框架获得完整戏剧结构。
- **结果**: 完成了一场结构完整的三场戏剧本，成为全书最长的综合练习案例。

### 案例 2: 戏剧结构讲习班的教学失误（c04）
- **问题**: 麦肯德里克第一次教授戏剧结构课时，让学生从电影大纲入手构思故事，结果几乎写不出漂亮的结局。
- **方法论的使用**: 他意识到大纲不应作为创意起点，应该回到故事构建框架本身——从"很久很久以前"的设定开始，逐步推进到结局，而非直接跳到结构分析。
- **结论**: 框架的顺序性（从设定到结局的逐步推进）比逆向分析更适合原创故事的开发。
- **结果**: 修正教学方法，从以大纲为起点改为以角色和故事框架为起点。

## A2 — 触发场景 (Future Trigger) ★
### 用户会在什么情境下需要这个 skill?
1. 用户有一个故事创意但不知道如何展开为完整结构——"我有个想法但不知道怎么写成完整的故事"
2. 用户写到中途发现故事失去方向——"我的故事到第二幕就散了"、"不知道怎么收尾"
3. 用户需要为游戏/互动媒体设计支线叙事——"每条支线任务都需要有完整弧线吗？"

### 语言信号
- "怎么构建故事结构"
- "故事写到一半不知道怎么继续"
- "怎么从一个想法变成完整的故事"
- "开头/结尾怎么设计"
- "故事框架"、"叙事模板"

### 与相邻 skill 的区分
- 与电影大纲分析法的区别：电影大纲分析法用于逆向拆解已有作品的结构骨架，本 skill 用于从零正向构建故事。
- 与 f04（戏剧反讽）的区别：戏剧反讽是框架中"为……所不知的是"一个步骤的具体展开，本 skill 是整体结构框架。

## E — 可执行步骤 (Execution)
1. **确定"很久很久以前"的时空设定和主角**
   - 完成标准：能用一句话说清"在什么时代/地点，有一个什么样的人"。主角有明确的身份标签。
   - 判停条件：如果设定模糊到无法想象具体画面，需重新选择。

2. **设计"但是"——主角的核心障碍或困境**
   - 完成标准：障碍是具体的、可感知的，而非抽象的"他很迷茫"。能回答"是什么在阻止主角得到他想要的？"
   - 判停条件：如果障碍只是外在的偶然事件而非与主角有因果关联，需调整。

3. **确定"有一天，事情发生了"——攻击点（故事真正开始的时刻）**
   - 完成标准：攻击点是一个具体事件，它打破了主角原有的生活平衡，迫使主角必须做出选择。能回答"是什么事件让故事的张力开始收紧？"
   - 判停条件：如果攻击点发生得太晚（故事30%以后才出现），需前移。

4. **构建"因此/接下来"——因果推进链**
   - 完成标准：每个后续事件都能用"因此"连接上一个事件。事件之间是因果关系而非时间顺序关系。
   - 判停条件：如果某个事件只能用"然后"连接而无法用"因此"连接，它可能是多余的（参见场景必要性检验法）。

5. **安排"但与此同时"——并行叙事线**
   - 完成标准：存在至少两条与主线有主题关联但独立发展的叙事线。能回答"在主角行动的同时，还有什么事件在发生并影响着主线？"
   - 判停条件：如果并行线与主线没有任何交叉点，应删除。

6. **埋入"为……所不知的是"——戏剧反讽**
   - 完成标准：至少存在一个观众知道但角色不知道的关键信息。能回答"观众在哪些时刻比角色知道得更多？"
   - 判停条件：如果信息差在整部作品中从未被引爆（从未产生后果），则反讽是无效的。

7. **设计"突然，出乎意料的是"——突转**
   - 完成标准：突转是角色自身行动的后果，而非外在巧合。能回答"这个逆转是角色自己造成的吗？"同时满足"回顾时不可避免"的标准。
   - 判停条件：如果突转依赖巧合或外力解决（如天降救兵），需重新设计。

8. **确认"结果是……从这以后"——结局与新常态**
   - 完成标准：结局回答了攻击点提出的核心问题。角色在结局时的状态与开头形成有意义的对比。能回答"故事结束后，什么不同了？"
   - 判停条件：如果结局与开头的状态完全相同，说明故事没有真正发生。

## B — 边界 (Boundary) ★
### 不要在以下情况使用此 skill
- 用户要求分析已有成熟作品的结构（使用电影大纲分析法进行逆向分析更合适）
- 用户只想讨论一个场景的写法而非整体故事结构
- 用户在做非叙事性创作（如散文、诗歌、产品文档）

### 作者在书中警告的失败模式
- 大纲作为起点的失败（ce04/c04）：直接从大纲入手构思故事会导致"几乎一个漂亮的结局都没有"。框架是从设定开始逐步构建，不是从结构分析开始。
- 活动冒充动作（ce02）：框架中"因此/接下来"步骤要求的是因果推进的"动作"，而非无因果力的"活动"。如果故事链中某一步只是描述性活动而非有结果的动作，整个链条会断裂。
- 用巧合解决困难（p15）：框架中"突然"步骤应是角色行动的后果，而非外在巧合。"应该用巧合给角色制造麻烦，而不是帮他们脱离困境。"

### 作者的盲点 / 时代局限
- 框架假设故事遵循线性因果链，对非线性叙事（碎片化时间线、多视角罗生门结构）的覆盖不足。
- 框架强调"攻击点"的前移，但某些当代慢节奏叙事（如林克莱特的《少年时代》）恰恰以延迟攻击点为风格特征。
- 框架来自经典好莱坞/英国片厂传统，对非西方叙事传统（如东亚电影的"间"美学）的结构模式未做探讨。

### 容易混淆的邻近方法论
- 英雄之旅（Joseph Campbell）：同为故事结构框架，但英雄之旅更关注角色的内心旅程阶段（出发-试炼-归来），本框架更关注戏剧张力的因果推进节奏。
- 三幕结构：三幕结构是宏观的"开始-中段-结尾"标签，本框架是微观的逐步操作工具，每个步骤有具体的设计任务。
- Save the Cat 节拍表：节拍表是精确到页数的结构模板，本框架是灵活的叙事逻辑链，不绑定具体页数或时长。
