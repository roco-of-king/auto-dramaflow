# 分区曝光系统

## R — 原文 (Reading)
> 入射读数、平均反射率18%、灰卡上点测光表的读数、第V区——以上表达的都是一回事，只不过是以不同的角度看问题而已。
> — Brown, 第10章, p.g_0042

## I — 方法论骨架 (Interpretation)
分区曝光系统（Zone System）由安塞尔·亚当斯发明，将亮度范围分为0-X区（11级），每区相差一档曝光：

- **第0区**：纯黑，无细节
- **第I区**：接近纯黑，略有质感
- **第V区**：中灰（18%反射率）= 入射测光读数 = 灰卡读数
- **第X区**：纯白，无细节

核心操作只有两步：
1. **定位（Place）**：决定画面中某个物体应该落在哪个分区，然后调整曝光使它出现在那里。
2. **跟随（Fall）**：一旦你将某个物体定位到目标分区，场景中所有其他物体将按它们与被定位物体之间的亮度差，自动落入对应的分区。

这个框架将曝光从"太亮了/太暗了"的技术判断，转化为"我希望这个物体看起来多亮"的创意决策。你不是在"正确曝光"，而是在"设计亮度关系"。

## A1 — 书中的应用 (Past Application)
### 案例：窗帘定位到第VIII区
- **问题**：用10K灯模拟阳光射入窗户，窗帘测光读数f/11。导演希望窗帘"很热但不过曝"。
- **方法论的使用**：将窗帘定位到第VIII区（比中灰高3档）。f/11往下数3档=f/4。镜头设定f/4，窗帘在成品中将是"很热的白色"。
- **结论**：定位让你精确控制"热到什么程度"——第VII区是浅灰，第VIII区是很热但有细节，第IX区是接近过曝。
- **结果**：演员站在f/2.8处（入射读数），跟随到第IV区（比中灰暗1档）。如果需要演员正常曝光，必须改变灯光比而非光圈。

### 案例：点测光表与分区转盘
- **问题**：场景中没有中灰物体，如何确定曝光？
- **方法论的使用**：在演员脸上测到EV10，典型亮色皮肤在第VI区。转动分区转盘让10对准第VI区，读取第V区的曝光值即为镜头应设光圈。
- **结论**：分区系统让你可以从任何已知亮度的物体反推曝光，而不依赖灰卡或入射测光表。

## A2 — 触发场景 (Future Trigger) ★
### 用户会在什么情境下需要这个 skill?
1. 用户想精确控制画面中某个元素的亮度——"我希望背景是浅灰色而不是白色"
2. 用户在高对比度场景中需要决定保留哪部分细节——"窗户过曝了怎么办？"
3. 用户想理解为什么测光表给出的读数和他想拍的效果不一致

### 语言信号
- "分区曝光"、"Zone System"、"第V区"
- "我想让这个更亮/更暗"、"背景应该是灰色的"
- "测光表读数"、"中灰18%"、"灰卡"
- "高光不过曝"、"暗部有细节"

### 与相邻 skill 的区分
- 与 hard-soft-light 的区别：硬光/软光影响的是对比度范围（亮暗差多大），分区系统决定的是"这个差值在成品中呈现为什么样子"。
- 与 three-point-lighting 的区别：三点布光解决"灯放哪里"，分区系统解决"最终看起来多亮"。

## E — 可执行步骤 (Execution)
1. **判断场景中关键物体的目标分区**
   - 亮色皮肤通常在第VI区，中灰物体在第V区
   - 白色但有细节的物体（如窗帘）在第VII-VIII区
   - 完成标准：能回答"这个物体在成品中应该有多亮？"

2. **用点测光表测量并定位**
   - 测量目标物体的读数，将其定位到目标分区
   - 计算偏移量：如果读数是f/8，目标是第VII区（比第V区高2档），则光圈应为f/4
   - 完成标准：光圈计算正确

3. **检查跟随结果**
   - 场景中其他物体将按亮度差落入对应分区
   - 如果跟随结果不满意（如演员太暗），需要改变灯光比而非光圈
   - 完成标准：所有关键元素的亮度关系符合叙事需求


   - 🔴 判停条件: 若定位后其他关键元素的跟随结果不在可接受的分区内（如演员落入第II区以下），需要改变灯光比而非继续调整光圈

## B — 边界 (Boundary) ★
### 不要在以下情况使用此 skill
- 用户只是想"正常曝光"而不需要精确控制亮度关系
- 用户在使用全自动模式拍摄（分区系统需要手动控制曝光）
- 用户在讨论后期调色而非前期曝光

### 作者在书中警告的失败模式
- 只改变光圈的陷阱：如果将窗帘定位到第VIII区后发现演员太暗，不能只调光圈（那会改变窗帘的定位），必须改变灯光比。
- 忽略跟随：定位一个物体后，必须检查所有其他物体的跟随结果，否则可能出现意料之外的亮度。
- 过度依赖测光表：测光表假设所有物体都是18%中灰，高反射率物体（白色婚纱）和低反射率物体（黑色西装）会欺骗测光表。

### 作者的盲点 / 时代局限

Blain Brown 的分析框架源自传统胶片摄影，对数字传感器的特性（如动态范围、高光滚降、暗部噪声）讨论不足。他假设摄影团队有充足的设备和时间，独立制作或低预算场景的应对策略覆盖较少。案例库以好莱坞和欧洲艺术片为主，对亚洲、拉美等非西方电影传统的摄影风格涉及有限。

此外，Zone System 源自安塞尔·亚当斯的胶片摄影体系，Brown 对其在数字传感器上的等效应用讨论不够深入。数字摄影的线性响应曲线与胶片的对数曲线有本质差异，直接套用 Zone System 可能导致曝光策略偏差。此外，他未充分讨论 LOG 拍摄模式对 Zone System 思维的替代作用。
