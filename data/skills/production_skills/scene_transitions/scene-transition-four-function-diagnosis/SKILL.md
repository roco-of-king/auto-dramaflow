# 场景转换四功能诊断法

## R — 原文 (Reading)

> 场景转换有四种功能：让观众思考、让观众感受、舒缓或形成张力、与人物产生联系。每次场景转换必须明确满足其中一种或多种功能。
> — Bays, 第1章, p.20

---

## I — 方法论骨架 (Interpretation)

这是一个场景转换的"决策分类器"。它的核心思想是：场景转换不是"怎么从A场景跳到B场景"的技术问题，而是"观众在这段间隙中应该经历什么心理功能"的设计问题。

四种功能分别是：（1）思考功能——通过信息揭示、闪回、文字等让观众的认知参与；（2）感受功能——通过强烈的情感场景、音乐、表演让观众的情绪被带动；（3）舒缓功能——通过景物、低潮、慢节奏让观众的紧张感得到释放；（4）联系功能——通过交通场景、运动中的主人公让观众与人物建立情感纽带。

这个工具的独特价值在于：它把"我觉得这个转场需要什么"的模糊直觉转化为"这个转场满足四种功能中的哪一种"的明确诊断。每种功能导向完全不同的技法选择——"思考"导向闪回/信息揭示，"感受"导向情感场景/音乐，"舒缓"导向景物/低潮，"联系"导向交通场景/运动中主人公。

## A1 — 书中的应用 (Past Application)

### 案例 1: 《角斗士》8分钟场景转换（c01）
- **问题**: 影片需要在竞技场场景和主角回忆之间进行多次转换，如何确保每段转换都有明确功能？
- **方法论的使用**: 用四功能诊断法逐一扫描：交通场景满足"联系"功能（观众跟随主角的身体移动建立情感连接），景物镜头满足"舒缓"功能（让观众从激烈战斗中恢复），闪回满足"思考"功能（让观众理解主角的内心世界）。不同段落分别满足不同功能，形成有机的转换节奏。
- **结论**: 8分钟的场景转换之所以不显冗长，是因为每段都有明确的功能诊断。
- **结果**: 这段转换成为全书最长的综合案例，证明四功能诊断法可以指导复杂转换设计。

### 案例 2: 末节场景的反面教训（ce01）
- **问题**: 很多新手导演的场景转换变成了"空洞填充物"——没有情感、没有信息、没有舒缓效果。
- **方法论的使用**: 用四功能诊断法扫描这些失败转换，发现它们的共同特征是"缺乏任何明确功能"——场景没有目的就变成了末节场景。
- **结论**: 末节场景产生的根本原因是缺乏四功能诊断。
- **结果**: 四功能诊断法既是预防工具（设计时使用），也是诊断工具（剪辑时使用）。

## A2 — 触发场景 (Future Trigger) ★

### 用户会在什么情境下需要这个 skill?

1. 用户在设计场景转换但不确定应该用什么风格——"这段转场应该用音乐还是用静默？"
2. 用户在剪辑时发现某段场景转换"不对劲"但说不清原因——"这段转场感觉很平，但不知道为什么"
3. 用户在设计游戏/动画/短视频的过场动画——"过场动画应该让观众做什么？"

### 语言信号 (用户的话里出现这些就应激活)

- "场景转换应该怎么做"
- "转场应该是什么风格/类型"
- "这段转场感觉不对"
- "过场动画/间场该用什么"
- "场景之间怎么衔接"

### 与相邻 skill 的区分

- 与末节场景诊断法(v09)的区别：末节场景诊断法是"治疗"——识别已有的问题场景并删除/修改；四功能诊断法是"预防"——在设计阶段就确定场景转换的功能。
- 与转化策略四象限(v03)的区别：转化策略四象限专注于音频维度（音乐/静默组合），四功能诊断法覆盖所有维度（视觉、听觉、叙事、情感）。

## E — 可执行步骤 (Execution)

当 skill 被激活后，agent 应按以下步骤执行：

1. **确定场景转换的"进入状态"和"退出状态"**
   - 完成标准：能回答"上一个场景结束时观众的情绪/认知状态是什么？"和"下一个场景开始时需要观众处于什么状态？"
   - 判停条件：如果用户无法描述进入/退出状态，需要先分析前后场景。

2. **用四功能诊断法扫描：转换需要满足哪种心理功能？**
   - 完成标准：明确选择一种或多种功能——思考（需要观众认知参与）、感受（需要观众情绪被带动）、舒缓（需要释放紧张感）、联系（需要与人物建立情感纽带）。
   - 判停条件：如果发现转换不需要任何功能（纯技术性连接），则直接跳到步骤4。

3. **根据功能选择对应技法**
   - 完成标准：选定的技法与功能匹配——思考→闪回/信息揭示/文字；感受→情感场景/强烈表演/音乐；舒缓→景物/低潮/慢节奏/静默；联系→交通场景/运动中主人公/跟踪镜头。
   - 判停条件：如果用户没有执行能力（如在剪辑阶段无法重拍），则提供可用素材中的最佳选择。

4. **验证转换是否"隐形"**
   - 完成标准：转换完成后，观众应该感觉不到"转场"的存在——功能的实现是无缝的，而非突兀的。如果转换让观众意识到"这里在转场"，需要调整。
   - 判停条件：如果用户刻意追求风格化转场（如黑场、闪白），则此标准不适用。

## B — 边界 (Boundary) ★

### 不要在以下情况使用此 skill

- 用户在讨论整体故事结构或剧本大纲（使用故事构建框架更合适）
- 用户在讨论单一场景内部的镜头调度（使用镜头设计相关skill更合适）
- 用户在做非叙事性视频（如教学视频、产品演示），这些视频的"场景转换"逻辑不同

### 作者在书中警告的失败模式

- 末节场景（ce01）：缺乏四功能诊断的直接后果——场景没有明确功能就变成了空洞填充物。
- 音乐的机械式使用（ce06）：将音乐当润滑剂而非有机组成部分——选了"感受"功能但用了错误的技法。

### 反模式与常见错误

- 强行给每个转换都分配功能——有些技术性连接（如时间跳跃的硬切）不需要任何心理功能，强行添加功能反而让过渡变得冗余
- 四功能混用导致观众心理状态混乱——同一个转换中同时追求"感受"（情绪爆发）和"舒缓"（释放紧张），两种功能相互抵消，观众既没有被情感冲击也没有得到放松
- 用四功能诊断替代对具体场景的情感判断——功能分类是决策工具而非创作本身，过度依赖分类会把"这段转换应该让观众流泪"简化为"这段转换满足'感受'功能"

### 作者的盲点 / 时代局限

- 贝斯的四功能分类基于传统线性叙事电影，对互动媒体（游戏、VR）的功能扩展有限。
- 四功能之间存在互补和冲突关系（如"感受"与"舒缓"不能同时达成），但书中对冲突处理的指导不够详细。
- 贝斯过度依赖好莱坞经典案例，对非西方叙事传统的场景转换功能分类未做探讨。

### 容易混淆的邻近方法论

- 三幕结构/节拍表：这些是宏观叙事结构工具，本 skill 是微观的场景转换决策工具。
- 剪辑节奏理论：剪辑节奏关注镜头层面的快慢，本 skill 关注场景层面的功能分配。
