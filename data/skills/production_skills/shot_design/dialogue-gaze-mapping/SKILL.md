# 目光方向空间映射框架

## R — 原文 (Reading)
> 克鲁尼目光望向一个方向，而房间里的其他人的目光都望向他。正是这些目光的方向让观众了解了每个人在房间中的位置。演员这种身体和目光的角度绝对不是无意而为之的。
> — Kenworthy, 第4章 §4.2, p.g_0020

## I — 方法论骨架 (Interpretation)
多人对话场景中最常见的问题不是"怎么切镜头"，而是"观众根本不知道谁在哪里"。传统的全景建立镜头可以解决这个问题，但它打断节奏、暴露空间、破坏紧张感。目光方向空间映射提供了一种更优雅的替代方案：用角色自己的目光方向作为观众的"GPS导航"。

核心机制很简单——主角望向房间（建立他是焦点），其他所有人望向他（通过视线汇聚告诉观众主角在哪里，以及其他人围绕他分布）。当一个新角色插话时，其他角色转身看他——这个转身动作同时交代了新说话者的位置和之前其他人的位置。

关键设计要点：(1) 目光方向必须是有意设计的，不能随演员自由发挥；(2) 有些角色的脸在画面中央，但目光偏向一侧——这种"偏斜注视"比正面注视更能突出凝视方向；(3) 身体和头的倾斜角度应与目光方向一致，形成"身体→目光→空间"的三重信息传递。

这个框架适用于所有多人对话场景，从三人到十人以上群戏都有效。它不仅能交代位置，还能暗示权力关系（谁被所有人注视=谁是焦点）和情感关系（谁在回避谁的目光）。

## A1 — 书中的应用 (Past Application)

### 案例 1: 《十一罗汉》十人群戏的空间清晰化
- **问题**: 超过10人参与的会话场景，需要让观众看清楚每个角色的方位
- **方法**: 乔治·克鲁尼背对墙站着，目光望向屋子——暗示他是焦点。所有其他演员的目光都指向克鲁尼，有些角色的脸在画面中央但目光偏向一侧，更突出凝视方向。低角度仰拍两名演员，他们头的倾斜角度一致，证明他们都在听克鲁尼讲话
- **结果**: 不需要全景镜头，观众就清楚地知道每个人的位置和对话流向

### 案例 2: 《十一罗汉》转身现人交代新说话者
- **问题**: 马特·达蒙一开始在背景中未说话，观众不清楚他的位置。他突然插话时，观众不知道他在跟谁说
- **方法**: 导演切到相反角度，让其他演员转身看达蒙——克鲁尼与达蒙有眼神交流，其他演员调整姿势看他。这既交代了达蒙的位置，也显示了其他人的位置
- **结果**: 每个人的反应都能看到，相互位置很清晰，对话双方的眼神交流也明确了

## A2 — 触发场景 (Future Trigger)

### 用户会在什么情境下需要这个 skill?
1. 拍摄多人对话场景时不知道怎么交代角色空间关系
2. 导演不想用全景镜头但需要让观众理解"谁在看谁"
3. 群戏场景中某个角色突然插话，需要让观众快速定位此人
4. 想在不打断对话节奏的前提下建立空间清晰度

### 语言信号
- "群戏太乱了，观众不知道谁在哪"
- "不想切全景，但需要交代位置"
- "多人对话拍得像一堆人在聊天，没有层次"
- "突然插话的角色，观众反应不过来"
- "谁在看谁交代不清楚"

## E — 可执行步骤 (Execution)

1. **设计焦点角色与目光网络**
   - 确定场景中的焦点角色（通常是主角或当前说话者）
   - 画出所有角色的目光方向线，确保它们汇聚于焦点角色
   - 完成标准：每个角色的目光方向都指向焦点角色或正在说话的人

2. **编排转身动作交代新说话者**
   - 当新角色开始说话时，让至少一个其他角色转身看他
   - 转身应同时显示转身者的原位置和新说话者的位置
   - 完成标准：观众能在不切全景的情况下，通过转身动作知道新说话者在房间的哪个方位

3. **用偏斜目光强化方向信息**
   - 部分角色的脸可放在画面中央，但目光偏向一侧
   - 头和身体的倾斜方向应与目光方向一致
   - 完成标准：观众能通过目光方向（而非角色位置）推断出所有人在房间中的空间关系


   - 🔴 判停条件: 若观众仍无法通过目光方向推断出所有人的空间关系，或转身动作未能同时交代转身者和新说话者的位置，回到步骤1重新设计目光网络

## B — 边界 (Boundary)

### 不要在以下情况使用此 skill
- 两人对话场景：空间关系简单，不需要目光映射，过肩镜头或双人镜头就够了
- 悬疑片中故意隐藏某人位置：如果剧本要求观众不知道某角色在哪里，就不应该用目光方向暴露
- 角色正在隐藏情绪的场景：如果角色需要"面无表情"或"假装没看见"，强制设计目光方向会破坏表演的真实感

### 作者在书中警告的失败模式
- 反例：如果只用全景和每人近景的切换方式，"会让整个情节变得很无趣"，而且"会让观众搞不清楚谁在看着谁说话"（§4.1）
- 反例：当对话已经进行了一会儿，有人突然插话，如果"先前没有交代这个人的位置，观众就会被搞迷糊了"（§4.1）

### 作者的盲点 / 时代局限
- Kenworthy 的案例全部来自好莱坞主流叙事电影，对非写实风格（如韦斯·安德森的对称构图）中的空间映射策略未做讨论
- 书中假设所有场景都需要"让观众知道谁在哪里"，但某些实验电影或艺术电影可能故意制造空间迷失感（如戈达尔后期作品），此时此框架不适用
- 目光方向设计在动画片中容易实现（角色完全可控），但在真人实拍中需要演员的高度配合，书中未讨论排练方法

### 容易混淆的邻近方法论
- "转身现人"（§4.1）和"目光示人"（§4.2）是同一框架的两种子技巧：转身用于交代新说话者，目光方向用于持续交代所有人的位置
- 全景建立镜头是传统的替代方案，但会打断节奏并暴露空间——此框架正是为了避免这种打断而设计的
